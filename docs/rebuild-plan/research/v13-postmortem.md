# v13 Architecture Post-Mortem — what to respec, redesign, drop

**Scope:** the tree at `extensions/job-application-tracker/v13` (13.0.0 → 13.0.2, commits `85617f1…cb25d19`), read in full before deletion. Verdicts feed the rebuild plan.
**Ground truth of failure:** 13.0.0 shipped as a fast fork of the v12 engine ("v13: fork v12 engine, full identity rebrand"); Pierre's first live run found it "nowhere close" — auto-apply inert, closing the window killed the brain, blank web dashboard, `{rows:{rows}}` documents, import fidelity wrong. All hotfixed in 13.0.1/13.0.2 (394 tests green), but the accretion problem stands → delete-and-rebuild ADR 2026-07-10.
**Headline judgment:** the *architecture* survived contact with reality almost everywhere it was actually exercised; the failures were **process failures in the seams the plan specced but the ship skipped** (checkpoints, scheduler, PatchBus, contract tests). Carry the designs; rebuild the process.

## 4. Top 10 structural rules for the rebuild (process, not features)

1. **A stage is done when Pierre has watched the checkpoint pass — never before.** Every stage ends in a user-witnessed, harness-evidenced demo; the next stage is blocked on his confirmation. (v13 shipped M1–M5 features with zero live checkpoints; the plan had them written and they were skipped.)
2. **Every pipeline ships whole: producer + consumer + one end-to-end test that moves real data terminal-to-terminal.** A queue with no enqueuer passed 300+ tests. The E2E test ("tracked application → terminal run state") would have failed 13.0.0 in CI.
3. **Dev-drive lands in stage 0.** The in-app loopback harness (drive + inspect + main-process screenshots) exists before the first feature, and every checkpoint's evidence comes from it. Observability of the real running app precedes code that needs observing.
4. **One API envelope, generated types, and a contract test per route.** Response shape is a convention enforced by CI against a seeded DB, not a per-route improvisation. Request bodies are zod-parsed. The renderer/popup consume generated clients — no hand-copied shapes or vocab (the `{rows:{rows}}` and "Saved"/"Tracked" drift both die here).
5. **Loud on unknown, everywhere a directory is scanned by convention.** A migrations file that doesn't match the naming regex, an adapter that fails schema parse at build time, an unroutable frame — each is a build/boot **error**, never a silent skip. (Runtime adapter hot-loads may degrade gracefully; packaged builtins may not.)
6. **No raw SQL outside the DAL — grep-gated.** `\.prepare\(` outside `db/dal/` fails CI. v13's law leaked within weeks (api.ts profiles/answers, run-service ledger); a law without a gate is a comment.
7. **No hardcoded identity, ever.** Ports, tokens, paths, product names flow from `@shared` constants + pairing storage; a grep-gate rejects literal port numbers and identity strings outside `shared/constants`. (The SW's hardcoded 7860 broke pairing the first time dev/prod split mattered.)
8. **Renderer modularity is a CI gate, not an aspiration.** Module-per-page, max-file-size lint (~300 lines), store with patch-only mutation, payload/boot budgets enforced against a seeded 5k-row DB from the first page onward. A 1,476-line main.js is a build failure.
9. **Deferred work carries an executable gate.** Anything postponed ("backup wired when 002 lands") is a skipped/failing test with the deferral in its name — visible in every CI run — never a prose comment. Equally: every structural law in the plan maps to a named milestone + CI gate, or the law is struck from the plan.
10. **Acceptance runs against reality.** The importer's sign-off is a dry-run on a copy of the live DB with per-section counts reviewed; the extension's sign-off is a live apply in real Chrome; Gmail's is a real OAuth consent. Synthetic fixtures gate regressions; they never grant "done." (Also encoded as: the first live LinkedIn submit happens in stage 1, not after five subsystems of breadth.)

## 5. One-line summary for the planning agent

Respec (proven): FSM-as-data + guarded DAL + CHECK-truth schema, runner + resume-by-reclassification + RunGateway seam, adapters-as-data format, sensor/actuator/SW thin extension with lease-epoch + dormant-default, answer ladder, ledger cap, Codex-CLI single-seam AI, importer laws, Gmail service, per-lane discovery, watch-and-learn triple redaction, shared contracts, dev-drive harness (day one), tray lifecycle + serve-dir static (post-fix forms). Redesign: scheduler (build the real one), ws-gateway run routing, API envelope + contracts, renderer modularity, PatchBus (build-or-strike), importer acceptance process, adapter lifecycle. Drop: galaxy/Aurora theme scope, the v12 pillar corpus as living spec. The failures were process, not architecture — the ten rules above are the actual deliverable.
